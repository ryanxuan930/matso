- **裁決事件為 E2E 明示 affordance**：真 kernel tick loop 未在 E2E 跑；stub 模式下令成功即
  同步發一則 EVENT 到 Redis stream（代表 order 進 ledger/裁決），供煙霧測試驗「下令→事件→
  WS→UI」全鏈路。真裁決事件由 kernel 產出（部署層）。
- **E2E Redis**：本機用既有 redis 的 DB 15（隔離、不污染）；CI 用 job 內 redis service。
