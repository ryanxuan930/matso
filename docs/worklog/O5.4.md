## 決策與陷阱
- **架構分工（延續 O5.3 慣例）**：comms 模組＝純確定性物理（鏈路預算 + mesh），插件化；
  Core 端消費 link states 並強制 §6.2 戰術後果。terrain 遮蔽附加損耗 + weather RF 衰減由
  Core 從各 client 填入 request（模組保持純、可離線測試）——真實接線同 O3.6/O5.3 為部署層。
- **mesh 連通模型**：每 faction 指定 command anchor（`is_command_node`）；品質量化兩級後，
  單位最終狀態＝**兩級子圖 `nx.has_path` 至任一錨點**：全 ONLINE 邊子圖可達→ONLINE、否則非
  OFFLINE 邊子圖可達→DEGRADED（含 DEGRADED 跳，鏈路只如最弱一環）、皆否（孤島）→OFFLINE。
  （比 widest-path 演算簡潔且等價。）無錨點 → 退化為最佳直接鏈路。
- **陷阱：VHF 自由空間 range 極大**——tx 30dBm/rx −100dBm 下 FSPL 到 50km 仍 ONLINE；真實
  鏈路死於地形遮蔽而非自由空間損耗。測試以注入 obstruction_db 控制三態（非靠距離），亦印證
  「terrain 遮蔽附加損耗由 Core 填」的設計正當性。
- **comms 為非硬依賴**（不像 terrain）：插件不可達→全 ONLINE 降級（不因基礎設施故障懲罰玩家），
  不 PAUSE session。與 weather 的 CLEAR 降級同理。查無單位 → ONLINE 樂觀預設（同一哲學）。
- **移除 config.py**：comms 純確定性、無外部資料來源，settings 為 dead config（jamming 由
  request 攜入，非啟動設定）。
- §6.3 EW（干擾/SIGINT）為 Phase 1.5：`jamming_db` 僅留公式佔位（預設 0），本卡不建 jammer 模型。
