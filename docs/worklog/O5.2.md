## 決策與陷阱
- **WeatherProvider 抽象**：SYNTHETIC/LIVE 共用介面（payload_at + is_stale），service/plugin 只依賴介面。
- **stale 狀態機**：refresh 成功更新格網 + 記 last_success；失敗保留最後有效值 + stale=true。首次成功前恆 stale。恢復即回 LIVE。
- **時鐘注入**：weather 為外部微服務（非模擬引擎），牆鐘合法；now 以參數注入 → stale/告警可確定性測試。
- **CWA 真 API 不測**：parse_stations 為純函數（fixture 驗）；HTTP 錯誤路徑以 monkeypatch httpx 驗；真實欄位對映（O-A0001-001）於部署以 API key 校準。
- **stale → 插件 DEGRADED**（SPEC §16.3）：payload.stale（Core 消費）與 plugin.health（DEGRADED）雙軌；Core 收 stale>30min 告警（stale_alert 提供門檻）。
- **api_key env 注入**（MATSO_WEATHER_CWA_API_KEY），絕不寫死。
