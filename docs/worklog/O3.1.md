## 決策與陷阱
- **狀態機為唯一權威**（state_machine.py）：所有 OrderStatus 轉移經 next_status 驗證；服務層 submit（PENDING→VALIDATED/REJECTED）與 cancel（→CANCELLED）皆走它。
- **物理預檢注入**（PhysicsGateway Protocol）：與 gRPC 細節解耦，測試用假 gateway 不需真 terrain server；真轉接為 TerrainGatewayAdapter。**紅線**：物理事實由 terrain 裁決，AI 永不介入；不可行立即 REJECTED。
- **不可行仍落庫 REJECTED**：供稽核/AAR；API 回 422 + code + details.order_id（precheck 快照在 order.precheck）。驗證階段失敗（語法/權限）則不落庫。
- **terrain down → 503**：物理預檢硬依賴；TerrainUnavailableError（http_status=503）冒泡。上層 session PAUSE 由 O2.5 HealthMonitor 負責。
- **schemathesis v3 不支援 OpenAPI 3.1**（FastAPI 產 3.1）→ 用 v4（`schemathesis.openapi.from_asgi`）。契約測試只斷言「不 5xx」（最有價值的：端點永不崩）。
- **issued_at_tick 佔位**：OrderService.tick_source 預設回 0；kernel↔API 整合（後續卡）時改讀活 SimClock。
- **ledger 指令序列重播（R10）**：以固定 ScriptedOrder 序列（代表 Ledger 記錄的 order 序列）驅動 kernel，證明同序列→同 stateHash。完整「讀 DB Ledger 重播」屬 O8.1。
- **ruff B008**：FastAPI `Depends()` 於參數預設是刻意慣例，extend-immutable-calls 放行。
