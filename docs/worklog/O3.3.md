## 決策與陷阱
- **k-ring 不漏保證**：k = ceil(max_range/edge)+1；edge < 格心距（√3×edge）→ 過度覆蓋 → 決不漏射程內目標。精確距離在 k-ring 存活者上再算（haversine）。用 grid_disk 不用 grid_distance（HOW_TO §8 跨 base cell 拋例外）。
- **確定性擲骰序列**：感測器、候選皆按 unit_id 排序；每個射程內敵對配對擲一次 → k-ring 與暴力全配對產生完全相同結果（測試強制驗證）。
- **faction-scope 後端強制（紅線）**：store.query 一律 `WHERE faction=`；無任何方法回跨陣營 contacts。ContactView **永不含 target_unit_id**（ground truth 連結）；番號/型號/陣營依 fidelity 逐級揭露。
- **API faction 暫由 query 帶入**：⚠ 不信任 client——O7.5 RBAC 落地後改由認證主體（SessionParticipant.faction）推導。後端隔離機制（service/store）已就位，換的只是 faction 來源。
- **偵測 vs 情報等級**：roll<p_detect 才成立 contact；等級由 p_detect 高低（＝條件好壞）分級——近/清晰→高等級。
- **敵我判定簡化**：faction 不同即為敵（O3.3）；聯盟分組（BLUE+ALLIED）待想定設定。
