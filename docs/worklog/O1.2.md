## 決策與陷阱（之後維護的人需要知道的 why）
- **hash payload 只含決定性欄位**（sessionId/seq/tick/eventType/initiatorId/targetId/weatherSnapshot/terrainModifier/reasoningChain/aiDecision/damageCalc），**排除 id（隨機 uuid）與 timestamp（牆鐘）**——否則 golden replay 無法重現相同 hash。id/timestamp 由 DB/model 預設產生，不入鏈。
- writer 與 verify 必須用**完全相同**的 canonical_event_payload 建構函式，否則自我驗證會失敗——集中一個函式，兩處共用。
- timestamp 由 model `server_default=func.now()` 產生，Python 不呼叫牆鐘（守紅線）。
- seq 從 0 起、per session 連續；writer 記憶體快取 tip=(last_seq,last_hash)，首次由 DB 查最大 seq 初始化（single-writer 前提下記憶體即權威；O1.3 Kernel 持有 writer 跨 tick）。
