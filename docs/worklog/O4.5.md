## 決策與陷阱
- **issuer 由 token 推導**（非 body）：SPEC §12 前端不可信。orders 端點加 bearerAuth，查
  SessionParticipant(user, session) → issuer_id；非參與者 → 403。前端因此不需帶 issuer。
- **E2E stub gateway**（env `MATSO_STUB_GATEWAY`）：precheck 回 feasible + 假 ETA——讓下令全
  流程於無 terrain 的 SQLite E2E 可跑（同 seed_dev_user 的測試 affordance，非改真物理引擎）。
