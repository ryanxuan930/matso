# O1.1 SimClock + DeterministicRNG

## 目標摘要
建立模擬時間與隨機數的兩個唯一來源，作為 P4 可重現性（相同 seed + 相同指令序列 → bit-identical 結果）的地基。
- `SimClock`：以整數 tick 推進的模擬時鐘，`sim_time_ms = tick × tick_rate_ms`，完全不依賴牆鐘。
- `DeterministicRNG`：以 (master_seed, stream_id) 派生的 numpy PCG64 產生器；不同 stream（adjudication/sensors/comms…）互相獨立，避免跨子系統耦合。

## 決策與陷阱（之後維護的人需要知道的 why）
- SimClock 只管「目前 tick」與換算 sim_time_ms；時間壓縮比例（0x/1x/…/60x）屬 Kernel 排程層，不放這裡（TASKS.md 明訂）。
- `advance()` 語意上「只能由 Kernel 呼叫」——Python 無法硬性強制，以 docstring 標註 + 設計上讓 now() 為唯讀來降低誤用；審查時須確認呼叫點只在 Kernel。
- RNG 種子派生：`SHA256(f"{master_seed}:{stream_id}")` → 256-bit int（`int.from_bytes(..., "big")` 固定 endianness）→ numpy PCG64。測試證實同 seed 同序列、跨實例可重現。
- `choice` 以「產生索引」實作而非 numpy `Generator.choice`，避免任意 Python 序列被強制轉 ndarray 改變型別。
- **陷阱（已解）**：驗收 grep 是字面比對，docstring 裡寫出被禁函式名會誤觸；紅線說明改用「牆鐘 API」等描述性字眼，不留字面 `datetime.now()`/`time.time()`。
- **範圍紀律**：checkpoint 用的 RNG state 序列化（get_state/set_state）屬 O1.5，此卡不做，已記入 PROGRESS backlog。
