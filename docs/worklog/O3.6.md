## 決策與陷阱
- **DoD 常駐 CI**：integration/conftest 的 session_factory 綁 MariaDB（無 compose 即 skip）。DoD 必須每次都跑 → 測試改用**本地 SQLite 引擎**（不取用該 fixture），於 python job 常駐、integration job 不重複（未標 integration marker）。
- **兩條 order 路徑**：MOVE 由 MovementSystem.step 自管（O3.4）；ENGAGE 走 kernel order_source→adjudicator。真實流程 drain（VALIDATED→EXECUTING）先於 resolve（EXECUTING→COMPLETED）——測試接線時勿跳過 drain（否則 VALIDATED→COMPLETED 非法轉移）。
- **接線只做 I/O 邊界**：EngagementAdjudicator 讀熱狀態 ammo/health/armor、呼叫純函數 resolve_engagement、寫回結果、轉移 order 狀態。物理裁決仍純函數，AI 不介入（紅線）。
- **測試武器 ph=1.0**：讓 DoD 傷害鏈確定性通過（命中率真實性由 O3.2 property 驗）；用真實 seed 武器會因 roll 而或中或不中，不利 DoD 斷言。
- **SENSOR_CONTACT 入 Ledger 是 ground truth 記錄**（White Cell/AAR 可讀）；前端投影去識別化由 IntelService 負責（分層）。
- **DB on async path**：接線的 sync DB 呼叫目前直接在 async drain/sweep 內；真實部署應以 asyncio.to_thread 包裝（HOW_TO §3.1）——沿用 O3.4 備忘。
