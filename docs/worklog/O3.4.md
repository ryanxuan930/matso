## 決策與陷阱
- **MovementSystem 自管 MOVE 生命週期**：step 內 admit（拉 VALIDATED MOVE 直接處理）+ advance，不走 kernel drain→adjudicator（避免把「移動」硬塞進「裁決」語意）。generic order 管線留給 engagement（O3.6）。
- **step 同 tick admit + advance**：order 於被 admit 的那個 tick 就開始移動（SPEC §2.3「下一個 tick 取出 pending」的 Phase-1 詮釋）。測試須注意這點。
- **from_h3 權威來自 DB**：DbOrderStore 由單位當前座標算 from_h3，不信任 client 給的起點。
- **single-writer**：位置/油料只由 MovementSystem（經 Kernel 呼叫 movement.step）寫入 hot_state。
- **中斷/油盡也結束 order（COMPLETED）**：execution 已完成（單位停下），事件區分 MOVE_INTERRUPTED/HALTED_FUEL。EXECUTING→COMPLETED 走狀態機。
- **油料為 stub**：fuel_per_hex 常數；hot_state 無 "fuel" 鍵時視為無限（不強制）。真實油耗表於 O5 logistics。
- **passable 為注入**：地形事件（某格變不可通行）由 passable 判定；真版可查 HexGridCache mobility / 動態障礙集。
