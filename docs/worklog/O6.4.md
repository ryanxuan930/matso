## 決策與陷阱
- **eval 量測模型原始輸出（護欄前）**，非 Gateway 後——§19.4「護欄前 ≤10%、護欄後 =0」。故 runner
  自行驗 schema/CoT/IHL/引用，不經 GuardrailGateway（避免把「清洗後」當「模型品質」）。
- **無真模型 → FallbackResponder**：產 schema-valid 佔位輸出（orders 空→不觸 IHL）。驗的是**管線**
  非模型品質；真模型 eval 為手動 workflow（O6.6）。§19.4 條件式 gate 正是為此。
- **空案例庫 → passed=True + EVAL_CORPUS_EMPTY**：不擋，但明確警告「綠燈≠戰術品質」。
- reasoning_chain 有 minLength 80（base schema）——FallbackResponder 與測試夾具的 CoT 都要夠長。
