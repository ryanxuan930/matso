## 決策與陷阱
- **detail 欄不入 hash chain**（刻意）：它承載非決定性診斷（overrun 牆鐘、rollback 中繼資料）。入鏈會破壞重播可重現性；不入鏈則 detail 可被竄改而不觸發 verify——取捨：detail 定位為「非證據性診斷」，證據性欄位仍全在 hash 內。文件明載。
- **R2 修法 = rollback 刪除較晚 checkpoint**：checkpoint 是狀態快取非證據（ledger 才是 append-only 證據），刪除被回滾世代的快照是正當的；tick ≤ target 的舊 checkpoint 是共同前綴、保留。
- **R3 修法 = checkpoint 錨定 ledgerSeq**（單調）取代 tick（回滾後非單調）；load_latest 依 ledgerSeq 排序。
- **R6 降頻規則**：interval = max(nominal, 上一 tick 實際耗時)，連續 overrun 時自然而然把節奏拉到可負荷；恢復即回額定。簡單、無震盪。
- **R9 修法 = asyncio.to_thread 包同步 driver**（非全面改 async driver）：Kernel 逐一 await，順序不變 → 決定性不受影響。全面 async 化留給真的量測到瓶頸時。
