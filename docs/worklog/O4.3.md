# O4.3 WS stream + Pinia store（useSessionStream：HELLO/last_seq 補償/RESYNC）

## 目標摘要
即時串流（SPEC §16.2、contracts/ws_protocol.md）：後端 WS 端點
`WS /api/v1/sessions/{id}/stream?token=<jwt>`——token 認證 + faction-scope 過濾 + 從 Redis
ring buffer（O1.4 已建，5000）補送 + **背壓斷線**（per-client queue 上限 1000，慢 client 斷線
而非塞爆）。前端 `useSessionStream` Pinia store：HELLO 帶 last_seq、WELCOME/RESYNC_REQUIRED
處理、STATE_DIFF/EVENT 累積、自動重連。**驗收：斷線重連補齊事件的整合測試；慢 client 被斷線**。

## 關鍵約束
- **範圍檢查非差值檢查**（O1.7/R7）：last_seq 不在 ring 現存 seq 範圍（缺口過大 / seq 倒退）
  → RESYNC_REQUIRED，client 走 GET /state。
- **背壓禁無限緩衝**（HOW_TO §8）：queue 溢出 → 斷線 + 要求重同步。
- faction 過濾 MUST 後端（前端不可信，SPEC §12）。
